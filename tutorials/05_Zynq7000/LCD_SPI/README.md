# LCD_SPI — SPI LCD Driver Projects for EBAZ4205 PS

English | [日本語](README_JP.md)

This folder contains two Vitis application projects that drive a **ST7789** SPI LCD (240×240 px)
from the Zynq-7000 Processing System (PS) on the EBAZ4205 board.

## Hardware

| Signal | EMIO Pin | Description |
|--------|----------|-------------|
| CS     | EMIO 54  | Chip Select (GPIO manual control) |
| CD     | EMIO 55  | Command / Data select |
| RES    | EMIO 56  | Hardware Reset |
| SCK / MOSI | PS SPI0 | Controlled by PS SPI hardware |

- **SPI Mode**: Mode 3 (CPOL=1, CPHA=1)
- **Clock prescaler**: CPU\_1x / 4
- **Pixel format**: RGB565 (16 bit/pixel, big-endian)
- **Resolution**: 240 × 240 pixels
- **UART baud rate**: 115200 bps (ps7\_uart)

## Directory Structure

```text
LCD_SPI/
├── lcd_spi_ps/          # Project 1: Raw SPI LCD driver (no GUI library)
│   └── src/
│       └── main.c       # SPI init, LCD init, color-bar / fill-screen demo
└── lcd_lvgl/            # Project 2: LVGL v9 GUI demo on ST7789
    └── src/

        ├── lv_conf.h    # LVGL configuration (v9.6.0-dev)
        ├── lvgl.h       # LVGL master header
        ├── lv_version.h # LVGL version header
        └── src/         # LVGL v9 source tree (core, widgets, fonts, …)
```

## Project 1 — `lcd_spi_ps` (Raw SPI Driver)

A minimal bare-metal application that demonstrates direct ST7789 control.

### Features

- GPIO initialization for CS / CD / RES via EMIO
- Hardware SPI initialization using `XSpiPs`
- ST7789 register initialization sequence
- Frame buffer (`frame_buffer[240][240]`) with bulk DMA-style SPI transfer
- **Color bar** demo: 8-color vertical stripes
- **Fill screen** demo: cycles through 8 solid colors

### Key Functions

| Function | Description |
|----------|-------------|
| `Lcd_Gpio_Init()` | Initializes EMIO GPIO pins for CS / CD / RES |
| `Lcd_Spi_Init()` | Configures PS SPI0 in master mode (Mode 3) |
| `Lcd_Init()` | Sends ST7789 initialization command sequence |
| `Address_set(x1,y1,x2,y2)` | Sets the drawing window (column + row address) |
| `LCD_ColorBar_2D()` | Draws 8-color vertical bars via bulk SPI transfer |
| `LCD_FillScreen(clr)` | Fills the entire screen with a single color |

## Project 2 — `lcd_lvgl` (LVGL v9 GUI Demo)

Adds the **LVGL v9** open-source GUI library on top of the same SPI driver.

### Features

- LVGL v9 (`lv_conf.h` configuration for v9.6.0-dev)
- Partial rendering mode with a 10-row line buffer (`lv_buf1`)
- Custom flush callback (`lvgl_flush_cb`) that writes row-by-row via PS SPI
- Animated demo UI:
  - Navy background
  - Title label ("LVGL v9 Demo")
  - Animated progress bar (0 → 100 → 0, 50 ms timer)
  - Color-shifting center box (RGB interpolation)
  - Numeric value label

### Key Functions / Callbacks

| Function | Description |
|----------|-------------|
| `lv_init_display()` | Creates LVGL display, sets flush callback and partial buffer |
| `lvgl_flush_cb()` | Transfers each rendered row to ST7789 via SPI |
| `lv_demo_setup()` | Builds the demo UI (background, bar, box, label, timer) |
| `lv_demo_timer_cb()` | 50 ms timer: updates bar value, box color, and label |

## How to Use

### Prerequisites

- **Vivado 2023.2** — used to generate the hardware platform (XSA) that exposes PS SPI0 and EMIO GPIO
- **Vitis Classic 2023.2** — used to build and run the application
- EBAZ4205 board with ST7789 240×240 LCD connected via the adapter board

### Steps

1. **Open Vitis Classic** and create a new workspace.
2. **Create a Platform Project** from the XSA file generated by the `05_Zynq7000` Vivado project.
3. **Create an Application Project**:
   - For the raw driver: select `lcd_spi_ps` sources (`src/main.c`).
   - For the LVGL demo: select `lcd_lvgl` sources (all files under `src/`).
4. **Add include paths** in the application BSP / project settings:
   - `lcd_lvgl/src` (for `lv_conf.h`, `lvgl.h`)
   - `lcd_lvgl/src/src` (LVGL source root)
5. **Build** the project (right-click → Build).
6. **Run / Debug** via JTAG or load to SD card.
7. Open a serial terminal at **115200 bps** to view log output.

### Expected UART Output

**lcd_spi_ps**:

```text
EBAZ4205 LCD Test with PS SPI
COLOR BAR
Fill Screen Color=0
Fill Screen Color=1
...
```

**lcd_lvgl**:

```text
EBAZ4205 LCD Test with PS SPI
LVGL initialized successfully!
```

## Notes

- The ST7789 uses **big-endian** byte order for RGB565 pixels. Both projects perform a
  byte-swap before sending pixel data.
- CS is controlled manually via EMIO GPIO (`FORCE_SSELECT` option is set in `XSpiPs`).
- The LVGL source tree under `lcd_lvgl/src/src/` is the upstream LVGL v9 library and
  should not be modified. Adjust only `lv_conf.h` for configuration changes.
